Команда `gotr self-test` выполняет локальную проверку окружения и доступности API.

## Что делает

- Проверяет наличие обязательных параметров и доступность API.
- Диагностирует типовые проблемы окружения до запуска боевых операций.
- Даёт быстрый сигнал, что CLI готов к работе в текущем окружении.

## Когда использовать

- Когда нужно получить предсказуемый CLI-поток для автоматизации.
- Когда важно сократить ручные действия и риск ошибок.
- Когда операция должна одинаково выполняться локально и в CI/CD.

## Примеры

```bash
# Базовая проверка
gotr self-test

# Подробный вывод
gotr self-test --verbose

# Проверка с альтернативным конфигом
gotr self-test --config ~/.gotr/config/default.yaml
```

## Полезные флаги

- `--json` для вывода в машинно-обрабатываемом формате.
- `--output` / `--save` для сохранения результата в файл.
- `--verbose` для детальной диагностики выполнения.
